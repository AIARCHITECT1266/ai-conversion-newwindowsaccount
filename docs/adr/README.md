# Architecture Decision Records (ADR)

Dieser Ordner enthält formelle ADRs für Re-Evaluationen von
Architektur-Entscheidungen, die in Spec-Dokumenten wie
WEB_WIDGET_INTEGRATION.md festgelegt wurden.

Der Prozess ist dort im Abschnitt "Änderungs-Protokoll für
Architektur-Entscheidungen" (bzw. "Umgang mit Architektur-Revisionen")
definiert.

## Dateinamens-Schema

NNN-<kurz>.md — z.B. 001-web-widget-channel-enum.md

## Regeln

- ADRs werden nach Freigabe durch den Project Owner nicht mehr editiert.
- Neue Erkenntnisse kommen in ein neues ADR, das das alte ersetzt.
- Claude Code darf keine ADRs selbständig schreiben. Nur Trigger melden.
- ConvArch entwirft und bewertet ADRs, gibt sie aber nicht frei.
- Der Project Owner ist die einzige Freigabe-Instanz.
